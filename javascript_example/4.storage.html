Web Storage 개요

서버와 관계없이 사용자(브라우저) 측에서 데이터를 저장할 수 있는 기능입니다. 데이터는 key - value 형태로 저장되며, key와 value는 항상 문자열이어야 합니다.

숫자나 객체를 저장하면 자동으로 문자열로 변환됩니다. 객체는 JSON.stringify()를 사용해야 올바르게 저장할 수 있습니다.

LocalStorage · SessionStorage

localStorage sessionStorage
구분localStoragesessionStorage
보관 기간영구적 (직접 삭제 전까지)브라우저 탭/창 종료 시 삭제
범위같은 도메인의 모든 탭 공유해당 탭에서만 유효
용도아이디 저장, 자동 로그인임시 폼 데이터, 페이지 상태
sessionStorage는 서버의 세션(Session)과 다릅니다. 브라우저의 탭 단위 저장소입니다.

사용법

setItem getItem removeItem clear

저장 — setItem

localStorage.setItem("username", "hong123");
▶ 저장 결과
key: "username" / value: "hong123"

조회 — getItem

const name = localStorage.getItem("username");
console.log(name);
▶ 실행 결과
"hong123"

삭제 — removeItem

localStorage.removeItem("username");
▶ 실행 결과
// "username" 키가 삭제됨

전체 비우기 — clear

localStorage.clear();
▶ 실행 결과
// 저장된 모든 데이터 삭제됨

객체 저장

JSON.stringify JSON.parse

Storage는 문자열만 저장할 수 있기 때문에 객체를 저장할 때는 JSON.stringify()로 문자열화하고, 꺼낼 때는 JSON.parse()로 다시 객체로 변환해야 합니다.

저장

const user = { id: "hong123", name: "홍길동", age: 25 };

localStorage.setItem("user", JSON.stringify(user));
▶ 저장된 값 (문자열)
'{"id":"hong123","name":"홍길동","age":25}'

조회

const raw  = localStorage.getItem("user");
const user = JSON.parse(raw);

console.log(user.name);
▶ 실행 결과
"홍길동"
JSON.stringify() 없이 객체를 그냥 저장하면 "[object Object]"라는 문자열이 저장되어 데이터를 복원할 수 없습니다.

아이디 저장

localStorage

로그인 시 "아이디 저장"을 체크하면 다음 방문 때 아이디 입력란에 자동으로 채워집니다.

코드

// 로그인 시 — 체크 여부에 따라 저장 or 삭제
if (saveChecked) {
  localStorage.setItem("savedId", id);
} else {
  localStorage.removeItem("savedId");
}

// 페이지 로드 시 — 저장된 아이디 불러오기
const savedId = localStorage.getItem("savedId");
if (savedId) {
  idInput.value = savedId;
  saveCheckbox.checked = true;
}

자동 로그인

localStorage

"자동 로그인"을 체크하고 로그인하면 다음 방문 시 로그인 상태를 복원합니다. 사용자 정보를 객체로 저장하고 페이지 로드 시 확인합니다.

코드

// 로그인 시 — 자동 로그인 체크 시 사용자 정보 저장
if (autoLoginChecked) {
  const user = { id, name: "홍길동" };
  localStorage.setItem("autoLogin", JSON.stringify(user));
}

// 페이지 로드 시 — 자동 로그인 정보 확인
const raw = localStorage.getItem("autoLogin");
if (raw) {
  const user = JSON.parse(raw);
  showLoggedIn(user); // 로그인 상태로 전환
}

폼 데이터 유지

localStorage

입력 중 실수로 새로고침해도 작성 내용이 사라지지 않도록 합니다. input 이벤트마다 저장하고 페이지 로드 시 복원합니다.

코드

// 입력할 때마다 저장
titleInput.addEventListener("input", () => {
  localStorage.setItem("draftTitle", titleInput.value);
});
textarea.addEventListener("input", () => {
  localStorage.setItem("draftContent", textarea.value);
});

// 페이지 로드 시 복원
const savedTitle = localStorage.getItem("draftTitle");
const savedContent = localStorage.getItem("draftContent");
if (savedTitle)   titleInput.value = savedTitle;
if (savedContent) textarea.value   = savedContent;

Cookie와의 차이점

Cookie와 Web Storage는 모두 브라우저에 데이터를 저장하지만 목적과 동작 방식이 다릅니다.

구분CookieWeb Storage
주요 목적서버에 전달할 클라이언트 정보 저장클라이언트 측 데이터 저장
서버 전송매 요청마다 자동으로 서버에 전송서버로 전송되지 않음
용량약 4KB약 5MB
만료만료 날짜 설정 가능localStorage는 영구, sessionStorage는 탭 종료 시
로그인 토큰처럼 서버에서 확인이 필요한 정보는 Cookie, 아이디 저장이나 UI 상태처럼 클라이언트에서만 쓰는 정보는 Web Storage를 사용하는 것이 일반적입니다.
