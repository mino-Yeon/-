나는 초보자를 위한 DOM 학습용 인터랙티브 웹페이지를 만들고 싶다. @3.DOM.html  해당 파일은 이전에 만든 실습인데 마음에 들지 않아서 안 쓸거야 하지만 안에 있는 이론적인 학습내용은 참고하도록 해
다른 파일들은 참고하지마


목표:
사용자가 DOM 구조와 classList의 동작 원리를
눈으로 직접 체감하도록 만드는 것이 목적이다.

핵심 컨셉:
- 기본 HTML 문서를 제공한다.
- 해당 HTML로 생성된 DOM 트리를 화면에 시각화한다.
- 사용자가 DOM을 조작하면
  1) 실제 화면
  2) DOM 트리 시각화 영역
  이 동시에 변경되도록 만든다.

⚠️ 매우 중요:
classList를 통해 CSS 스타일이
적용되었다가 해제되는 것이 시각적으로 극명하게 보여야 한다.

따라서 반드시 기본 CSS를 포함하라.
CSS는 class 추가/제거 시 명확한 시각적 변화가 발생하도록 설계하라.

반드시 포함할 CSS 클래스:

- .highlight → 배경 노란색
- .active → 두꺼운 파란 테두리
- .hidden → display: none
- .important → 글자 빨간색 + bold
- .selected → 배경 파란색 + 흰색 글씨
- .fade → opacity 감소 + transition 효과

----------------------------------------

[초기 HTML 구조 요구사항]

다음 조건을 반드시 만족해야 한다:

- id가 있는 요소 포함
- 여러 class가 동시에 적용된 요소 포함
- 부모-자식-형제 관계가 명확한 구조
- ul > li 구조 포함 (노드 추가/제거 실습용)
- 빈 div 컨테이너 포함 (createElement 실습용)
- data-속성을 가진 요소 포함
- depth 3 이상 중첩 구조 포함
- classList.toggle 실습이 자연스럽게 가능한 구조

----------------------------------------

[학습 단계 설계]

1단계: 조회 실습
- querySelector / querySelectorAll
- parentElement / children / nextElementSibling
- getAttribute / dataset
- 선택된 노드를 .selected 클래스로 강조 표시
→ 화면과 트리 시각화에서 동시에 강조되도록 구현

2단계: 조작 실습
- textContent 변경
- 속성 수정
- classList.add / remove / toggle
→ 스타일 변화가 즉시 보이도록 구현

3단계: 노드 추가 실습
- createElement
- appendChild
- innerHTML
→ 추가 시 .fade 애니메이션 효과 적용

4단계: 노드 제거 실습
- remove()
- 특정 조건에 따라 제거 (예: data-id가 짝수인 li 삭제)
→ 트리 시각화에서도 즉시 반영

----------------------------------------

[UI 구성]

- 좌측: 실제 렌더링 화면
- 우측: DOM 트리 시각화 영역
- 하단: 실습 버튼 영역(코드를 직접 작성해서 실행할 수 있는 영역이면 좋을 듯) 

----------------------------------------

[추가 실습 기능]

- 버튼 클릭 시 모든 .item 요소에 .highlight 추가
- 다시 클릭 시 .highlight 제거
- 특정 조건에 맞는 요소에만 .important 적용
- 여러 class를 동시에 추가/제거하는 실습 기능 포함
