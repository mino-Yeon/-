비동기 통신 모듈이 필요한 이유

HTML에는 서버와 통신할 수 있는 태그가 이미 있습니다. 하지만 이 방식에는 한계가 있습니다.

<a>, <form> 태그의 한계

구분 <a> / <form> fetch (비동기 통신 모듈)
처리 방식 동기 — 응답 올 때까지 대기 비동기 — 응답 기다리지 않고 다음 실행

페이지 전환 없이 데이터만 주고받는 방식을 AJAX(Asynchronous JavaScript and XML)라고 합니다. 과거에는 XHR로 구현했지만, 현재는 fetch가 표준입니다.

fetch 기본 문법

fetch Promise

fetch(url)은 서버에 요청을 보내고 Promise를 반환합니다. 응답이 오면 .then()으로 결과를 받아 처리합니다.

fetch("https://httpbin.org/delay/2")  // Promise 반환
  .then(res  => res.json())           // 응답을 JSON으로 변환 (이것도 Promise)
  .then(data => console.log(data));   // 실제 데이터 사용

async / await으로 쓰면

async function request() {
  const res  = await fetch("https://httpbin.org/delay/2");
  const data = await res.json();
  console.log(data);
}

.json()도 Promise

res.json() await

fetch()가 반환하는 Promise가 fulfilled되면 Response 객체를 받습니다. 이 객체 안에 실제 데이터가 있는 게 아니라, 데이터를 꺼내려면 .json()을 한 번 더 호출해야 합니다.

.json()도 Promise를 반환하기 때문에 await을 두 번 써야 합니다.

async function getUsers() {
  const res = await fetch("https://dummyjson.com/users");
  console.log(res);        // Response 객체
  console.log(res.json()); // Promise — 실제 데이터가 아님!
}
async function getUsers() {
  const res  = await fetch("https://dummyjson.com/users"); // Response 객체
  const data = await res.json();                            // 실제 데이터
  console.log(data); // { users: [...] }
}

fetch() → Response 객체 → .json() → 실제 데이터, 이 두 단계를 기억하세요.

async 함수는 Promise를 반환

async Promise

async 함수를 호출하면 반환값이 무엇이든 항상 Promise로 감싸져서 반환됩니다. 그래서 async 함수 호출 결과에 .then()을 붙일 수 있습니다.

async function request() {
  // 2초 딜레이 서버에 fetch 요청
  const res  = await fetch("https://httpbin.org/delay/2");
  const data = await res.json();
  return data; // Promise 로 반환됨
}

const result = request();
console.log(result); // Promise {} — 아직 응답 안 옴

// 실제 데이터를 꺼내려면 .then() 또는 await 필요
request().then(data => console.log(data));
▶ 데모 — async 함수 반환값 확인 (2초 딜레이 서버)

실행 직후 result를 찍으면 Promise {<pending>}이 출력됩니다.
2초 뒤 응답이 오면 .then() 콜백이 실행됩니다.

버튼을 눌러 실행하세요.
이것이 실습에서 getUsers().then(users => { ... })가 동작하는 이유입니다. getUsers는 async 함수이고, async 함수는 항상 Promise를 반환하기 때문입니다.

리팩토링 — XHR → fetch

fetch async/await

Promise 학습에서 작성했던 getUsers(), getPosts(), getComments()를 XHR에서 fetch로 교체합니다. 코드가 얼마나 간결해지는지 비교해보세요.

getUsers()

function getUsers() {
  return new Promise((resolve, reject) => {
    const xhr = new XMLHttpRequest();
    xhr.open("GET", "https://dummyjson.com/users", true);
    xhr.onload = () => resolve(JSON.parse(xhr.responseText).users);
    xhr.onerror = () => reject("네트워크 오류");
    xhr.send();
  });
}
async function getUsers() {
  const res  = await fetch("https://dummyjson.com/users");
  const data = await res.json();
  return data.users;
}

getPosts(userId)

function getPosts(userId) {
  return new Promise((resolve, reject) => {
    const xhr = new XMLHttpRequest();
    xhr.open("GET", `https://dummyjson.com/posts/user/${userId}`, true);
    xhr.onload = () => resolve(JSON.parse(xhr.responseText).posts);
    xhr.onerror = () => reject("네트워크 오류");
    xhr.send();
  });
}
async function getPosts(userId) {
  const res  = await fetch(`https://dummyjson.com/posts/user/${userId}`);
  const data = await res.json();
  return data.posts;
}

getComments(postId)

function getComments(postId) {
  return new Promise((resolve, reject) => {
    const xhr = new XMLHttpRequest();
    xhr.open("GET", `https://dummyjson.com/posts/${postId}/comments`, true);
    xhr.onload = () => resolve(JSON.parse(xhr.responseText).comments);
    xhr.onerror = () => reject("네트워크 오류");
    xhr.send();
  });
}
async function getComments(postId) {
  const res  = await fetch(`https://dummyjson.com/posts/${postId}/comments`);
  const data = await res.json();
  return data.comments;
}
XHR 방식은 Promise 생성자, XMLHttpRequest 객체, onload/onerror 콜백이 모두 필요했지만, fetch를 사용하면 3줄로 동일한 동작을 구현할 수 있습니다.
